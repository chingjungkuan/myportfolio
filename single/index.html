產品服務｜可快速製作的產品會讓用戶使用 Single 編輯器，快速完成一本簡單的作品，與 Editor2 定位較不同。

Role | 🧑🏻‍💻 UIUX Designer

Collaborators | UIUX Designer(1)、PM(1)、FE(2)、BE(1)、CS(2)

在優化 Single 編輯器的專案中，我發現使用者常在響應式裝置上卡關，主要是介面缺乏明確的操作引導，讓人難以順利開始編輯，許多用戶因此在初期就放棄。我整合用戶回饋與競品分析，建立 Personas 和 Empathy Map，並與 CS 團隊釐清回報背後的真正需求。多數使用者並非抗拒編輯，而是不知道從哪開始、該怎麼做才對。


根據這些洞察，我調整資訊架構、簡化選項，設計出「任務導向」的分階段流程，讓使用者能專注一件事、一步步完成作品，降低中途離開的可能。 在流程設計與原型測試期間，我與 PM 密切合作，釐清各節點的功能與進入條件。最終新流程成功讓棄單率下降 25%。這次經驗也讓我更有意識地在設計中維持「操作上的明確感」，幫助使用者更有信心完成任務。

痛點與觀察 Pain Point & Task

舊版的問題

🔺過度依賴上下滾動
需頻繁滑動才能切換功能，讓使用者難以快速找到所需選項，降低操作效率。
🔺編輯區與選項分離
預覽畫面與操作選項分離，使用者需來回滾動確認效果，操作不夠直覺。
🔺關聯操作分散各處
像「新增」、「選物件」、「加雕刻」在上方，照片池與新增按鈕卻藏在下方，導致操作流程斷裂、學習成本高。
🔺預覽體驗不穩定
滾動切換功能時，作品位置容易偏移和抖動，干擾細節調整與即時預覽。
🔺行動裝置操作負擔大
手機螢幕小，加上需要多次點擊與滑動，增加操作難度。
🔺資訊層級不清
樣式、顏色、雕刻等選項未明確區分，容易讓使用者搞不清楚目前編輯狀態。

設計優化重點 Solution

    為了解決上述問題，我重新規劃了操作流程與介面結構，讓整體體驗更清晰舒適：
  • 統一操作區塊位置：將照片池固定於左側，並整合編輯工具列，減少視線與操作的跳轉。
  • 優化照片新增流程：使用者可直接於編輯區上傳與替換照片，流程更簡潔。
  • 重整按鈕與工具排列邏輯：依使用場景分類，提升操作直覺性。
  • 提升視覺一致性與精緻度：使用低飽和色系、加強層次感（圓角、陰影等），打造更現代的介面風格。
  • 調整字體與元件間距：改善閱讀與點擊體驗，提升整體易用性。

資訊架構 IA Define

先盤點現有功能，統整規格並解決不一致之處，建立規格文件與資訊架構。同時讓團隊成員快速理解產品結構，加速開發決策。

User Flow Ideate

精簡流程並重新梳理編輯體驗

當專案目標明確後，我從使用者角度出發，盤點現有編輯流程中的操作痛點與介面問題，列出需要優化的功能項目，例如：將「儲存」與「進階編輯」收合至 Header 精簡操作視覺，或重新評估「變換產品」功能在 Single 編輯器中的優先順序，以聚焦使用者最常進行的核心操作。

針對這些調整，逐步重構主要操作流程，並延伸繪製各功能的使用路徑圖，協助團隊更清楚掌握全貌。每個版本都會與團隊夥伴反覆討論，根據回饋持續優化，最終收斂出貼近實際使用情境且技術可行的設計方案。

從操作流程到線框圖，逐步落實設計邏輯

針對功能調整，我先重構主要操作流程，並繪製各功能的使用路徑圖，協助團隊掌握整體邏輯。每一版本都與團隊反覆討論、修正，逐步收斂成符合使用情境與技術可行的設計方案。完成資訊架構後，我會先在平板上進行簡單構圖，再用 Figma 繪製無色彩干擾的線框圖，針對流程調整較多的功能項目，確認並聚焦元件佈局與資訊層級。確認初步方向後，再製作精緻版本供團隊確認後才進入視覺設計階段。

UI Design

新版｜視覺風格簡約清楚且統一

📌界面層次更清晰

  • [舊版] 選單、按鈕、編輯區域的界線不夠明確，使用者可能需要花時間理解操作區域與編輯內容的關係。
  • [新版] 固定的編輯區塊、統一的按鈕樣式，讓功能層級更清晰，操作更直覺。

📌更專注的編輯區域

  • [舊版] 由於編輯內容隨滾動變動，使用者難以長時間專注於作品本身。
  • [新版] 透過固定編輯範圍，使用者的視線能夠集中在作品上，不會因滾動而影響專注度。

📌操作區塊的視覺統一性

  • [舊版] 不同的選項區塊間的間距、圖示大小不一致，導致視覺層級不夠統一。
  • [新版] 選項區塊的樣式更加一致，提升整體美感與操作流暢度。

📌更突出的按鈕與操作指引

  • [舊版] 某些按鈕較小，或是沒有明顯的標示，使用者可能需要額外花時間尋找功能。
  • [新版] 透過調整按鈕樣式與收折次要功能，讓使用者更容易聚焦當下操作，例如更清楚地引導他們完成「預覽」與完稿流程。

📌配色與視覺焦點更均衡

  • [舊版] 顏色對比度較低，某些選單可能容易被忽略。
  • [新版] 透過更具對比度的配色，將區塊明顯區分，讓用戶明確知道有三個區塊分別可做哪些操作。

📌更符合行動裝置使用習慣

  • [舊版] 按鈕與選單可能因滾動而偏離操作範圍，導致使用體驗下降。
  • [新版] 固定底部導覽列的形式，使用者可以輕鬆用拇指點擊選項，符合行動裝置的使用習慣。

作品編輯區

-整體

強化響應式：固定高度範圍

  • 明確定義不同段點區間裝置的畫面呈現
  • 優化後的設計更加直覺、層次分明，並且加強了視覺焦點與操作便利性，使得使用者在編輯過程中能更專注於作品本身，帶來更流暢的體驗。
-照片池

將上傳照片區從底部移至左側，強化編輯流程與空間配置

設計策略：依據產品特性調整空間規劃。

針對卡片、木框畫、海報等編輯步驟簡單、圖片張數少（通常都不超過 5 張）的產品，我們設計了輕量化的 Single 編輯器，用於支援快速完成創作的使用情境，與功能更完整的 Editor2 作出區隔。 因此在空間配置上，我們做出以下調整：
將上傳照片區從底部移至左側側欄，利用較小區塊呈現，避免干擾主要編輯區操作。
中央區域專注於編輯畫面預覽與內容調整，強化視覺焦點與操作效率。
右側配置為外觀樣式與設定選項，確保樣式變化與編輯區能同時對照，縮短來回切換時間。
-導覽列

Header 結構調整：聚焦編輯任務，簡化導覽選項

原本的編輯畫面直接沿用官網 Header，但其內容包含行銷活動與非編輯相關資訊，容易造成使用者分心、影響編輯流程的專注度。
呼應 Single 編輯器的設計目標 —— 支援快速編輯與下單，我們希望使用者能順暢完成從編輯 → 預覽 → 購買的流程，不被其他資訊干擾。
此外，若使用者點擊 Header 嘗試返回官網，系統會先透過Modal 確認用戶是否離開當前編輯畫面，避免誤操作導致資料遺失。

👉 具體做法
    這次調整將 Header 聚焦於「編輯任務」本身：
  • 移除與編輯無關的選項，或改為收折呈現。
  • 編輯狀態下，以「預覽」為主要按鈕。
  • 進入預覽後，以「完稿並購買」為主要 CTA。
-變換產品

變換產品：聚焦同系列，降低誤用風險

原本的版本提供完整產品列表，展開讓使用者自由滑動切換。但實際觀察發現大多數使用者傾向在同系列產品內進行變換，較少出現跨品類（如木框畫換成相片筆記本）的情境。

因此，我們調整為：
  • 僅提供「同系列產品」的變換選項，降低誤用率，也讓選項更聚焦。
  • 將變換流程設計得不那麼輕易觸發，避免干擾一般用戶的編輯流程。
  • 保留變換產品的彈性，讓有需要的使用者仍可快速沿用照片內容，套用至相似產品中，提升轉換的便利性。
  • 兼顧功能的可用性與界面的簡潔性，也避免讓不必要的選擇成為操作上的干擾。
-Tooltips 教學

增加友善的教學 Tooltips：即時引導、降低學習門檻

針對特定產品的部分功能（如海報 與 木框畫 的「新增雕刻」），我們加入情境式 Tooltips，在使用者初次進入編輯器適時提示，協助理解操作方式，降低學習門檻。 此機制保留彈性，未來新增功能也能沿用，不影響版面配置，提升功能可見性與使用率。

－功能 Bar

介面工具列優化：提升操作效率，統一互動邏輯

頁面功能 Bar 優化：強化使用體驗
  • 將功能列移至編輯區上方，操作更直覺。
  • 優化體驗，減少滑動與視線移動。
  • 功能分區明確，有助快速完成編輯。

樣式與操作一致化：文字與照片工具列同步更新
  • 統一文字與照片工具列樣式，符合新版設計系統。
  • 文字 Modal 支援字體、字級、顏色與對齊設定。
  • 無需回外層調整，Modal 內即可完整編輯。

功能命名與行為邏輯統一
  • 維持一致命名與開啟邏輯，降低使用者認知負擔。將原有不同名稱的設定按鈕（如「編輯作品」、「基本內容」…）統一為「作品資訊」。
  • 作品資訊 Modal 中統一可編輯項目為：作品名稱與作品說明。
Style Guide

📌 視覺層次更清晰，資訊組織更有條理

    [1.0]
  • 介面元素較密集，顏色較重，資訊層次不夠明顯，且視線需要來回移動，增加認知負擔。
    [2.0] 採用更明確的「分區」設計。
  • 左側邊欄：照片池管理功能統一集中，操作邏輯更直觀。
  • 主要編輯區：留白更多，內容層級更清楚，讓使用者可以專注於編輯工作。
  • 工具欄區塊分明：上方選單與左側功能區塊排列更一致，減少雜亂感。
  • 外觀樣式區：根據不同產品對應可編輯項目，用戶能快速辨識並選擇合適的樣式。

📌 使用圓角、留白與陰影，增加設計細節與質感

    [1.0]
  • 版面偏向早期網頁設計風格，較多銳利邊角與硬性分隔，缺乏柔和的視覺層次。
    [2.0]
  • 使用圓角設計，如按鈕、卡片、選單，讓介面更友善與現代化。
  • 適當留白，提升整體的呼吸感，減少壓迫感，讓用戶更容易專注在主要內容上。
  • 細膩的陰影與分層，例如照片編輯區與工具欄之間的視覺區隔更清晰，避免資訊混亂。

📌 色彩與對比更簡潔，符合現代 UI 設計趨勢

    [1.0]
  • 深色背景（深灰＋橘棕色）加上舊有風格是透過漸層創造立體感，但對比強烈容易造成視覺疲勞，且功能區塊之間的區隔不夠明顯。

    • [2.0]
  • 採用更柔和、現代的配色（淺米色 + 細緻的灰色 + 低飽和橘色），整體視覺更溫暖、親和，符合當代 UI 設計趨勢。
  • 背景顏色變淺，讓內容與圖片更突出。
  • 對比度優化，讓視覺更柔和但仍保持清晰可讀性。

📌 字體設計與排版更精緻，提升可讀性

    [1.0]
  • 按鈕與文字區塊稍顯擁擠，字體大小與間距略小，導致可讀性較差。
    [2.0]
  • 標題更明顯，字距更適中，將關聯性較高的內容集中在同區塊，使用更細緻的字體與留白。
  • 按鈕與功能名稱間距拉開，減少擁擠感，提高可讀性。

Takeaway

💡 建立清晰的資訊架構，讓內容更有層次、操作更順暢

原以為越多資訊越清楚，實際上過多細節反而造成干擾。精簡後，使用者反而更快找到重點。

💡 問對問題，才能真正解決使用者的困擾

把「繼續編輯」、「加入購物車」、「作品欣賞」...等關鍵動作按鈕直接放在卡片上，讓動作成為視覺焦點，提升使用者對下一步的確定感。

💡 如何提升跨部門溝通效率並精準傳達設計思維

透過清楚的設計邏輯與流程圖，幫助 PM 與工程團隊迅速掌握重點，降低溝通成本並減少反覆修改，使設計更貼近產品節奏與目標。
儘管過程中曾有觀點差異，雙方仍能理性溝通並理解彼此立場，這也成為我在跨部門協作中最深的學習。
