Design Impact

主要操作 CTA 點擊率提升

將「繼續編輯」等 關鍵行動按鈕直接置於作品卡片上，使用者能更快識別並執行操作，有效降低思考與點擊成本，進而提升 CTA 點擊率達 45%。

簡化步驟和資訊，操作效率提升

當附加功能收合後，用戶無需跳轉至其他頁面即可完成操作。根據 用戶回饋與行為數據顯示，操作步驟減少後，使用流程更為直覺，轉換率提升了 15%，用戶平均完成時間也縮短了 30%。

客服相關問題回報量下降

簡化介面後，用戶更容易理解作品狀態與後續操作，客服關於「作品去哪了」、「如何進入編輯器」的 詢問明顯減少。

痛點與觀察 Pain Point & Task

🗯️ 資訊層級混亂，重點不易辨識

  • 資訊列表的文字缺乏層級差異，所有資訊呈現方式相似，難以一眼抓到關鍵內容。
  • 顯示過多「次要資訊（如圖片數量、檔案大小）」，反而分散注意力，使用者 需額外花時間篩選重要資訊。

🗯️ 操作路徑繁瑣，效率低落

  • 缺乏明確的「主要動作（如「繼續編輯」）」指引，使用者需進入詳細頁才能操作，流程冗長。

  • 操作選項如「複製」、「刪除」 需進入下一層畫面才能執行，對有大量作品的使用者來說操作成本高。

🗯️ 介面佈局擁擠，視覺負擔大

  • 作品卡片圖片比例過大且垂直排列導致資訊密集，空間使用不佳。

  • 卡片切割區塊過多、缺乏留白與視覺節奏，使使用者在瀏覽時感到壓迫與混亂。

🗯️ 行動指引不清，主次不分明

-「繼續編輯」、「作品欣賞」等行動按鈕樣式不夠明顯 ，容易與周圍資訊混淆。
- 行動區缺乏層次感與視覺突出效果，重要操作按鈕位置偏下且需捲動，易被忽略。

設計優化重點 Solution

雖然本次專案未進行大規模問卷與訪談，我們仍透過使用數據、客服回饋及競品觀察，歸納出使用者在作品操作流程中常遇到的問題，並以此作為設計優化的切入點。
為進一步釐清方向，我分析多個業界競品的作品管理介面，整理其操作邏輯與版面結構的優劣，歸納出共通的使用行為與設計模式。

根據這些洞察，重新規劃資訊架構與操作動線，聚焦提升使用效率與資訊清晰度，優化整體體驗。

資訊架構Define

為了讓產品在擴充功能或優化介面時具備一致且清晰的結構，先對盤點現有功能，釐清現行設計中可能存在的混亂或規格不一致問題。接著將功能邏輯整理為資訊架構與規格文件，幫助團隊更快理解整體產品脈絡與使用者操作邏輯。

User Flow Ideate

讓流程問題具象化的第一步

我先繪製了 UI Flow，將頁面路徑與層級視覺化，協助團隊辨識舊版的痛點，例如操作層級過深、「繼續編輯」需多次點擊才能到達對應頁面，以及按鈕主次層級不明確。藉由這些洞察，我們優化了介面結構與資訊架構，使層級更清晰、流程更直覺。

Wireframe+Lo-fi Prototype

簡化開發流程 驗證操作假設

在設計初期，我們以低擬真原型快速建立可互動的 Wireframe，便於團隊溝通元件配置與操作邏輯，同時初步驗證流程是否簡化、能否降低使用者思考成本。

這次優化聚焦於移除不必要資訊，僅在需要時透過「詳細資訊 Modal」顯示內容，使介面更簡潔、重點更聚焦。

UI Design

統一新版設計風格

  • 風格更貼近品牌個性
    新版介面採用柔和配色與簡約排版，傳遞溫暖、小清新的品牌氣質，營造親和、舒適的使用氛圍。

  • 分層概念
    使用了卡片式設計，將內容明確區分（如左側側邊欄、作品列表區），讓使用者直覺理解資訊架構。

  • 簡化視覺
    採用柔和配色與低對比，減少過度裝飾，依內容需求排版，營造舒適且聚焦的閱讀體驗。

  • 一致性與突出層次
    標題、選單、圖像區塊明確，資訊層次感提升讓介面更直覺易懂。

主次分明，強調核心資訊

[Before]
資訊密集缺乏重點強調，使用者需要額外搜尋才能找到關鍵內容。

[After]
透過 Tab 區分作品狀態（如「編輯中」、「已完成」），每張卡片聚焦呈現重點資訊：清楚的標題、縮圖與狀態對應資訊標示。 輔助操作如複製、刪除、分享、詳細資訊...等則收納於 More 選單中，減少畫面負擔，無須再進入第二層頁面。

👉 具體做法
  • 使用不同字重區分主次資訊，例如作品名稱與狀態強調呈現，建立日期與編號則作為輔助資訊。
  • 卡片依據狀態顯示不同內容，例如「編輯中」的作品會提示棕色字的截稿日期。臨近期限時則以紅色字標示，強化視覺提醒。

按鈕優先級明確，行動導向清楚

[Before]
缺乏明確操作按鈕，使用者需點擊作品卡片並進入下一層頁面才能執行操作，容易忽略或增加操作時間。

[After]
每張作品卡片 根據狀態不同，皆設有明顯的主要行動按鈕，如編輯中的作品是「繼續編輯」，以完稿的作品是「加入購物車」可直接操作，無需跳轉頁面，提升使用效率。

👉 具體做法
  • 主要 CTA 使用高對比色，與背景形成明顯視覺區隔。
  • 簡化畫面資訊，讓用戶可以更聚焦在不同作品卡片上。

More 選單設計，互動方式更直覺

[Before]
用戶需點擊作品卡片才可以看到動作按鈕，且所有選項都直接顯示在頁面上，增加視覺負擔且干擾理解主要操作。 [After]
More 按鈕（三個點圖示）讓額外功能像是複製、刪除...等，不佔用主要空間，但仍然容易存取。

👉 具體做法
  • 次要功能收納在 More 按鈕中，保持畫面簡潔。
  • More 選單使用簡潔的圖標與文字，確保易讀性與可操作性。

空間感優化，讓畫面更易讀

[Before]
資訊擠在一起，沒有適當的留白，導致閱讀壓力較大。 [After]
每個作品卡片的資訊排列整齊，不同產品類型的區塊間距適當，讓整體畫面更乾淨易讀。

👉 具體做法
  • 增加適度的留白並確保不同資訊區塊有明顯分隔。
  • 讓卡片的文字和圖片內容均衡分布，瀏覽時更直覺。
Style Guide

📌視覺層次更清晰，資訊組織更有條理

[Before] 元素密集且顏色偏重，缺乏層級引導，使用者視線需來回瀏覽，增加負擔。

[After] 提升卡片與背景的對比度，並運用醒目色引導使用者聚焦操作行為，使畫面閱讀更輕鬆直覺。

📌 色彩與對比更簡潔，符合現代設計趨勢

[Before] 桃紅與深灰的強烈對比讓視覺較為尖銳，區塊分隔不明確，卡片切割過細，降低整體閱讀性。

[After] 採用柔和、低飽和的配色（細緻灰＋蜜桃橘），營造溫潤親和的視覺氛圍。同時強化區塊結構，更符合現代設計。

📌 字體設計與排版更精緻，提升可讀性

[Before] 文字與按鈕區塊略顯擁擠，字級與間距偏小，降低閱讀舒適度。

[After] 透過更清晰的標題層級、適當的字距與留白，提升整體排版節奏；同時將關聯內容歸納在同一視覺區塊，強化資訊結構與視線引導。

📌 圓角、留白與陰影，強化介面質感與親和力

[Before] 版面風格偏早期網頁設計，邊角銳利、分隔生硬，缺乏視覺層次與柔和感。

[After] 透過圓角應用於按鈕、卡片與選單，營造更友善且現代化的風格；搭配適度留白與陰影，提升界面呼吸感與整體質感，使用者更易聚焦於主要內容。

Takeaway
整體設計過程以使用者操作為核心，首先聚焦在「簡化」。

💡 使用者其實不需要那麼多資訊

原以為越多資訊越清楚，實際上過多細節反而造成干擾。精簡後，使用者反而更快找到重點。

💡 將關鍵動作放在「對的位置」，減少用戶多想

把「繼續編輯」、「加入購物車」、「作品欣賞」...等關鍵動作按鈕直接放在卡片上，讓動作成為視覺焦點，提升使用者對下一步的確定感。

💡 資訊結構重整帶來清爽感

優化後的畫面更整齊、分明，不少使用者回饋「比較好找」、「不會眼花」，證明視覺層次的影響力很大。

💡 選擇不該是壓力，而是指引

畫面上不需要一次呈現所有可能，反而要在適當時機給出明確建議。讓使用者在不同狀態都知道「現在該做什麼」，附加功能做收合，減少認知負擔。
